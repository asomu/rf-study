# Phase 01 Plan — GSM Burst / PvT Lab

## Goal

Deliver one complete learning loop that moves an RF learner from concept orientation to prediction, simulation, fault diagnosis, and teach-back evidence without presenting an educational approximation as a conformance system.

## Execution Order

1. Lock the M1 learning contract, source boundary, UI contract, and deterministic model.
2. Build the responsive application shell and route structure.
3. Implement Learn diagrams and prerequisite diagnostic.
4. Implement Lab prediction gate, parameters, chart, margin evaluation, and explanations.
5. Implement six Challenge cases and evidence-based diagnosis.
6. Implement Review competency matrix, saved Teaching Snapshot, print layout, and explicit teach-back completion.
7. Verify model fixtures, component behavior, end-to-end persistence, 1440/768/375 responsive behavior, WCAG-impacting violations, and print bounds.
8. Complete the GSD 6-pillar visual audit and independent final review.

## Acceptance Evidence

- Deterministic golden scenarios cover baseline, timing, ramp, overshoot, ringing/settling, and off-region floor.
- Every chart declares units, reference, educational-mask status, first violation, and worst margin.
- Parameter changes are not evaluated until a learner predicts the outcome.
- Challenge answers require both diagnosis and trace evidence.
- Review distinguishes concept, prediction, judgment, diagnosis, and teach-back.
- No page-level horizontal overflow at the three target widths.
- No axe serious/critical violations on the four routes.
- Teaching Snapshot fits an A4 landscape viewport.

## Non-goals

- Official 3GPP conformance limits or verdicts.
- Calibrated instrument correlation.
- Backend accounts, cloud sync, or real measurement import.
